# C-MAPSS RUL 项目实施计划

日期：2026-06-28  
目标：固定论文方向，拓展多模型实验矩阵，并把仓库整理为可公开推进的科研/复现双用途 repo。

## 1. 固定定位

论文工作题目：

> Safety-Oriented Model Selection for Aero-Engine RUL Prediction: Ranking Discordance and Risk-Profile Shaping on C-MAPSS

核心研究问题：

> 在 NASA C-MAPSS FD001-FD004 上，当评价标准从整体 RMSE 扩展到临近失效误差、过度乐观预测风险、不确定性和维护决策代理后，RMSE-best 模型是否仍然是最合适的模型选择？

当前项目的双重出口：

- 进：作为学生科研成果，形成一篇会议论文或 workshop / student research paper。
- 退：作为 NASA C-MAPSS RUL 多模型复现 repo，提供规范数据协议、模型矩阵、评价指标和论文表图生成脚本。

创新边界：

- 创新点不是“提出全新万能架构”，而是 safety-oriented model selection。
- Paper 1 是主线：评价协议、排序分歧、SARBI、RMSE 与风险指标的 trade-off。
- Paper 2 / Dual-LSTM 是扩展：验证 risk-profile shaping 是否能在同一协议下改变过度乐观风险。
- 不宣称航空安全认证、不宣称真实机队泛化、不宣称 SOTA。

## 3. IMRaD 论文结构

### Introduction

要回答的问题不是“哪个模型最低 RMSE”，而是“在安全偏好和维护风险下，模型选择是否改变”。引入 C-MAPSS、RUL、晚期预测误差、过度乐观预测的维护风险，以及现有文献偏向架构竞赛的问题。

### Methods

必须讲清楚：

- FD001-FD004 数据协议：engine-level validation split、train-only scaling、RUL cap、window size、last-window test。
- 模型矩阵：classical ML、deep sequence、safety-weighted losses、Dual-LSTM、后续 TCN/Transformer/UQ。
- 指标：RMSE、MAE、NASA S-score、critical RMSE30/50、overestimation ratio、overestimation magnitude。
- SARBI：只作为 declared-preference reporting index，不作为物理安全公式。
- 统计与复现：multi-seed、bootstrap、敏感性分析、脚本化输出。

### Results

按排序分歧组织，而不是按模型宣传组织：

- Representative matrix：不同指标选择不同模型。
- RMSE-best vs risk-best：FD001-FD004 的 winner split。
- Safety loss ablation：风险指标改善通常伴随 RMSE 成本。
- Dual-LSTM extension：cycle consistency 是 modest representation signal，safety weighting 更明确地减少 overestimation magnitude。
- 后续扩展模型加入后，要报告“排序分歧是否仍存在”，而不是只报告新模型是否赢。

### Discussion

强调：

- Aggregate RMSE 不足以作为唯一模型选择依据。
- SARBI 的价值是让评价偏好透明化。
- C-MAPSS 只是 simulated benchmark，不能等同真实发动机健康管理。
- 如果 TCN/Transformer 取得更低 RMSE，但在风险指标上排序仍分裂，这反而强化论文主张。
- 如果强模型同时改善 RMSE 和风险指标，也要如实报告；这不会削弱 repo 价值，反而提高复现基准可信度。

### Conclusion

一句话结论：

> C-MAPSS RUL 模型选择应同时报告整体精度、临近失效误差和过度乐观风险；在现有多模型结果中，这些评价目标经常产生不同排序，因此 safety-oriented model selection 是比单一 RMSE 排名更稳健的报告方式。

## 4. 必须降调的结论

- “Safety-GRU / Dual-LSTM 更安全”降为“在 C-MAPSS benchmark risk metrics 上降低 optimistic overestimation 或 critical-zone error”。
- “SARBI 选出最安全模型”降为“SARBI 是透明的相对报告指标，必须与 raw metrics 和 sensitivity 一起读”。
- “Safety loss 提高模型性能”降为“Safety loss 改变 risk profile，常伴随 subset-dependent RMSE cost”。
- “Cycle consistency 提升 RMSE”降为“cycle consistency 提供 modest representation signal；多数 RMSE bootstrap interval 未排除 0”。
- “Dual-LSTM 是 SOTA / first-ever”必须删除，改为“method-response candidate”。
- “Bootstrap 证明真实安全收益”必须删除，改为“paired seed-engine robustness check on simulated C-MAPSS”。
- “深度模型优于 classical baselines”必须删除；Paper 1 反而显示 classical baselines 很强。

## 5. 模型拓展路线

### Tier 0: 已实现基线

用途：稳定复现、生成论文当前证据。

- Ridge
- Random Forest
- Gradient Boosting
- optional XGBoost
- LSTM
- GRU
- 1D CNN
- Safety-GRU / safety-weighted losses
- Cycle-consistent Dual-LSTM

验证重点：

- `metrics.csv`、`predictions.csv`、`training_history.csv`、`selected_features.csv` 输出 schema 不变。
- `scripts/make_safety_benchmark_outputs.py` 和 Paper2 输出脚本继续可用。

### Tier 1: TCN / Dilated Temporal CNN

用途：补足 C-MAPSS 文献中常见且比当前小型 CNN 更强的序列基线。

建议实现：

- 在 `src/rul_prediction/models_deep.py` 或新增 `src/rul_prediction/models_temporal.py` 增加 `tcn` 分支。
- 在 `src/rul_prediction/train_deep.py` 的 `--models` choices 中加入 `tcn`。
- 在 `scripts/run_research_matrix.py` 和 deep ablation 脚本中允许 `tcn`。
- 新增最小测试：模型 forward shape、smoke training 2 epochs。

优先实验：

```powershell
python scripts\run_research_matrix.py --subsets FD001 FD004 --seeds 42 43 44 --deep-models tcn --deep-epochs 60 --skip-existing
```

### Tier 2: Attention / Transformer Encoder

用途：检验更强表示模型下 ranking discordance 是否仍然存在。

建议实现：

- 先做 lightweight Transformer encoder，不做大模型化。
- 加 positional encoding 或 cycle index embedding。
- 保持输入输出 schema 与现有 deep models 一致。
- 不把 Transformer 写成主创新，只作为强基线。

优先实验：

```powershell
python scripts\run_research_matrix.py --subsets FD001 FD002 FD003 FD004 --seeds 42 43 44 --deep-models transformer --deep-epochs 80 --skip-existing
```

### Tier 3: Uncertainty / Calibration

用途：把点预测扩展到风险意识决策支持。

可用路线：

- MC dropout：低成本，适合先跑。
- Deep ensemble：成本高，但更稳健。
- Conformal interval / quantile regression：适合作为后续补充。

已有入口：

```powershell
python scripts\run_uncertainty.py --subset FD001 --method mc_dropout --model gru --loss safety_mse
```

需要补充的指标：

- PICP 80/90/95
- MPIW / Winkler score
- critical-zone PICP30/50
- uncertainty-error correlation
- overconfident-overestimation ratio

### Tier 4: Decision Proxy

用途：把“安全偏好”从指标偏好推进到维护决策代理。

已有入口：

```powershell
python scripts\run_decision_simulation.py --predictions <prediction-files> --out-dir reports\tables\decision
```

后续要固定：

- lead time: 10, 20, 30, 50 cycles
- preventive cost
- early replacement cost per cycle
- late/missed-critical penalty
- SARBI-best 与 RMSE-best 的 policy cost 对比

### Tier 5: Robustness / Domain Shift / XAI

用途：增强复现 repo 价值，作为论文 Discussion 或 supplement 证据。

已有入口：

```powershell
python scripts\run_sensor_robustness.py --subset FD001 --model gradient_boosting
python scripts\run_domain_shift.py --source-subset FD001 --target-subset FD003 --model gradient_boosting
```

优先级：

- P1：sensor noise / masking robustness。
- P2：FD001 -> FD003、FD002 -> FD004 domain shift。
- P2：feature importance / XAI 用于解释模型脆弱性，不作为主贡献。

## 7. 实验优先级

### P0: 立即稳定当前仓库

- README 重写为公开版。
- 新增本实施计划。
- 保留已编译 PDF：Paper1、Paper2 main、Paper2 supplement。
- 忽略 LaTeX build/source 中间产物。
- 跑 `pytest`，提交并推送 main。

### P1: 最小可发表会议版

- 保持当前 Paper1 + Paper2 evidence。
- 在论文中合并成一条主线：evaluation protocol + risk-profile shaping extension。
- 增加 decision proxy 小节。
- 至少补一个强 baseline：TCN。
- 对 SARBI 权重、critical threshold、RUL cap 做敏感性分析。

### P2: 强化复现 repo 版

- 加 compact Transformer encoder 或 Attention-GRU。
- 完整 uncertainty/calibration 输出。
- robustness/domain shift matrix。
- 统一生成 `reports/model_cards/` 或 `reports/benchmark_cards/`，记录每类模型结果。

### P3: 更高目标投稿版

- 加更严格统计检验。
- 加维护策略模拟的 cost curve。
- 加 uncertainty-aware decision proxy。
- 若有真实或半真实外部数据，再讨论泛化；否则不越界。

## 8. 验收标准

每次新增模型必须满足：

- 支持 FD001-FD004。
- 支持 seed 参数。
- 不破坏 train-only scaling 和 engine-level validation split。
- 输出 schema 与现有聚合脚本兼容，或同步更新聚合脚本和测试。
- 至少有一个 smoke test。
- 论文表述必须同时报告 RMSE 和风险指标。

```powershell
python -m pytest -q -p no:cacheprovider
git status --short --untracked-files=all
```


## 9. 风险与约束

- C-MAPSS 是 simulated benchmark，不是真实机队遥测数据。
- “Safety-oriented”只指 late-life error 与 optimistic-overestimation benchmark metrics，不等于航空安全认证。
- SARBI 是偏好透明化工具，不是认证指标。
- Paper 1 是 evaluation benchmark / model-selection study，不是新架构 SOTA。
- Paper 2 是 method-response / risk-profile shaping study，不是 Paper 1 baselines 的通用替代品。
- 模型拓展不能变成无边界架构堆叠；每个新增模型都要服务于“排序分歧是否仍存在”。
- GPU 结果回传后必须先做 schema / seed / subset 完整性检查，再写入论文。
